## 总体概况

Ontology v1 是《史记》知识库的第一个稳定版本，通过 AI Agent 将《史记》130篇文本转化为结构化知识图谱。

### 核心特点

- **知识粒度化**：将长篇古文拆解为 675 个独立知识单元（SKU）
- **多类型组织**：事实性知识、技能知识、关系知识三类互补
- **实体增强**：7,497 个实体标注，覆盖 17 类命名实体类型
- **结构化存储**：JSON + Markdown 混合格式，便于机器处理与人类阅读
- **可追溯性**：每个知识单元都记录原文来源（chunk ID）

### 版本信息

- **创建时间**：2025年3月
- **来源文本**：《史记》全文 130 篇
- **总字符数**：约 62.7 万字（原文）
- **处理方法**：基于 Claude 的 AI Agent 自动提取 + 人工审核

## 知识单元类型

### SKU（Standard Knowledge Unit）概念

**SKU = Standard Knowledge Unit（标准知识单元）**

将《史记》的知识拆解为标准化、可管理的独立单元。每个 SKU：
- 有唯一 ID（如 `sku_001`, `skill_001`）
- 有元数据（分类、来源、字数）
- 有标准化的内容格式
- 可独立存储、检索、组合
- 可追溯到原文来源（chunk ID）

### 三大类型

| 类型 | 英文名 | 数量 | 用途 | 文件格式 |
|------|--------|------|------|----------|
| **事实性知识** | Factual SKUs | 434 | 记录史实、人物、事件、数据 | `header.md` + `content.json/md` + `entities.json` |
| **技能知识** | Procedural SKUs | 241 | 提取可复用的方法、流程、策略 | `header.md` + `SKILL.md` |
| **关系知识** | Relational SKU | 1 | 构建知识图谱（标签、术语、关系） | `label_tree.json` + `glossary.json` + `relationships.json` |

### 文件结构

每个事实性 SKU 目录包含 3 个文件：

#### 1. `header.md`（元数据）

```markdown
# sku-id-or-name

- **Classification**: factual
- **Source**: 史记_chunk_XXX
- **Characters**: NNNN

Brief description of content
```

#### 2. `content.json` 或 `content.md`（正文）

- **JSON 格式**：适合列表、结构化数据（如人物名录、年表）
  ```json
  {
    "title": "标题",
    "items": [
      {"name": "项目1", "description": "说明"},
      {"name": "项目2", "description": "说明"}
    ]
  }
  ```

- **Markdown 格式**：适合段落叙事、传记、评论
  ```markdown
  ## 标题

  正文内容，可能包含多个段落...
  ```

#### 3. `entities.json`（实体标注）

```json
{
  "entities_by_type": {
    "人名": ["张三", "李四"],
    "地名": ["长安", "洛阳"],
    "官职": ["丞相", "太尉"]
  },
  "entity_count": 15,
  "top_entities": [
    {"name": "张三", "count": 5},
    {"name": "李四", "count": 3}
  ]
}
```

### 数据来源

- 每个 SKU 的 `Source` 字段标注了原始来源（如 `史记_chunk_042`）
- chunk 是原文切分的片段，便于溯源和验证

### 文件结构

每个技能 SKU 目录包含 2 个文件：

```markdown
# skill-name

- **Classification**: procedural
- **Source**: 史记_chunk_XXX
- **Characters**: NNNN

```

#### 2. `SKILL.md`（技能详解）

标准化的 SKILL 文档格式，包含：

```markdown
# 技能名称

## Context（情境）
何时使用此方法

## Goal（目标）
要达成的效果

## Methodology（方法论）
具体步骤或决策框架

## Example（案例）
史记中的原始案例

## Analysis（分析）
成败因素、适用条件

## Modern Relevance（现代启示）
对当代的借鉴意义
```

### 技能的价值

- **可迁移性**：从古代场景抽象出通用方法
- **可操作性**：提供决策框架和步骤指导
- **可验证性**：有史记原文案例支撑

**Relational SKU（关系标准知识单元）**用**标签体系、术语表、实体关系三元组**构建的知识图谱，将分散的 SKU 组织为网络。

**文件**：`skus/relational/label_tree.json`

**结构**：层级分类树，20 个顶层分类

```json
{
  "label": "史籍",
  "children": [
    {"label": "纪传体"},
    {"label": "通史"},
    {"label": "编年体"},
    {"label": "八书"}

}
```

8. 朝代（10）：夏商周秦汉...
9. 建筑（4）：宫殿、陵墓、防御工程
10. 礼乐（4）：礼、乐、祭祀

**文件**：`skus/relational/glossary.json`

**结构**：978 条术语，每条包含定义、标签、来源

```json
{
  "term": "史记",
  "definition": "中国历史上第一部纪传体通史，由西汉司马迁撰写...",
  "labels": ["史籍", "纪传体", "通史"],
  "source": "太史公自序"
}
```

**示例术语**：
- 史记、司马迁、黄帝、帝尧、帝舜
- 禅让、郡县制、合纵连横、推恩令
- 鸿门宴、背水一战、毛遂自荐、完璧归赵
- 冒顿单于、和亲、张骞、西域

**用途**：构建史记专业词典，支持术语查询和定义

#### 3. 实体关系（Relationships）

**文件**：`skus/relational/relationships.json`

## 使用指南

### 1. 快速查找知识

#### 按主题浏览

1. 打开 [facts_index.md](facts_index.md) 或 [skills_index.md](skills_index.md)
2. 找到感兴趣的主题分类（如"军事战略与战术"）
3. 点击 SKU 链接跳转到具体内容

#### 按关键词搜索

```bash
# 搜索所有包含"背水"的技能
grep -r "背水" skus/procedural/

# 搜索所有提到"秦始皇"的 SKU
grep -r "秦始皇" skus/factual/*/content.*
```

#### 按实体查找

```bash
# 查找所有提到"张良"的 SKU
jq -r 'select(.entities_by_type.人名 | contains(["张良"])) | .id' skus/factual/*/entities.json
```

### 2. 知识图谱查询

#### 查找某人的所有关系

```bash
# 查找司马迁的所有关系
jq '.[] | select(.subject == "司马迁")' skus/relational/relationships.json
```

#### 查找因果链

```bash
# 查找"导致秦亡"的所有原因
jq '.[] | select(.object == "秦朝灭亡" and .predicate == "causes")' skus/relational/relationships.json
```

### 3. 批量处理

#### 统计实体频次

```bash
# 统计所有 SKU 中出现最多的人名
jq -r '.entities_by_type.人名[]' skus/factual/*/entities.json | sort | uniq -c | sort -rn | head -20
```

#### 提取所有技能名称

```bash
# 提取所有技能的标题
jq -r '.skus[] | select(.type == "procedural") | .title' skus/skus_index.json
```

### 4. 应用场景

#### 场景 1：教育培训

- **用途**：提取历史案例、领导力案例
- **方法**：查找技能 SKU，结合事实 SKU 作为案例支撑
- **示例**：培训"危机管理"时，使用 `skill_111`（功成身退）+ `sku_177`（范蠡传）

#### 场景 2：AI 应用开发

- **用途**：构建史记问答系统、对话机器人
- **方法**：
  1. 基于 `glossary.json` 构建术语库
  2. 基于 `relationships.json` 构建知识图谱
  3. 基于 SKU 内容训练检索增强生成（RAG）模型

#### 场景 3：学术研究

- **用途**：史记数字人文研究、社会网络分析
- **方法**：
  1. 基于 `entities.json` 构建人物共现网络
  2. 基于 `relationships.json` 分析因果链
  3. 基于技能 SKU 研究决策模式

#### 场景 4：内容创作

- **用途**：历史写作、故事创意
- **方法**：
  1. 从 [eureka.md](eureka.md) 获取跨领域灵感
  2. 从 [mapping.md](mapping.md) 按功能查找相关 SKU
  3. 组合多个 SKU 构建叙事
