# 谥号索引需求说明

## 概述

在专项索引中创建谥号索引页面，系统展示《史记》中出现的所有谥号及使用该谥号的君主信息。

## 数据结构

每个谥号条目包含以下信息：

### 1. 谥号字
- **示例**: 献、武、文、桓、惠等
- **格式**: 单字显示，作为条目标题

### 2. 谥号意义
- **来源**: 引用《谥法解》或其他谥法文献
- **示例**:
  - 献：聪明睿智曰献，智质有圣曰献
  - 武：克定祸乱曰武，刑民克服曰武
  - 文：经纬天地曰文，道德博闻曰文

### 3. 不同级别的谥号
按爵位等级分组：
- **天子级**: 献帝
- **诸侯王级**: 献王
- **诸侯公级**: 献公
- **诸侯侯级**: 献侯
- **大夫伯级**: 献伯
- **大夫子级**: 献子

### 4. 具体人物
每个级别下列出具体使用该谥号的人物：

#### 必需信息
- **人名**: 如"秦献公"、"晋献公"
- **链接**: 指向实体索引中的人名页面
- **出现章节**: 该人物主要出现的章节

#### 可选信息（如果已知）
- **在位年限**: 如"前384年-前362年"
- **生卒年**: 如"前424年-前362年"
- **备注**: 如"春秋五霸之一"

## 谥号识别逻辑

### 1. 谥号字符集定义

根据《谥法解》等文献，谥号分为三类：

#### 美谥（褒扬性）
```
文、武、成、康、昭、穆、宣、懿、献、景、惠、襄、烈、显、元、章、明、睿、
庄、定、简、靖、威、敬、安、桓、僖、孝、平、顷、恭、静、德、仁、肃、质、
贞、节、敏、缪
```

#### 平谥（同情性）
```
怀、悼、哀、闵、思、殇、幽、共、愍、夷
```

#### 恶谥（贬斥性）
```
厉、炀、灵、荒、隐
```

### 2. 爵位等级系统

按照周代爵位制度，分为七等：

```
帝 > 王 > 公 > 侯 > 伯 > 子 > 男
```

**注意**：
- "君"不是正式爵位，不应作为谥号识别标准

### 3. 人名模式识别

#### 基本模式
```
[前缀?] + [谥号字] + [爵位]
```

- **前缀**：国名、氏族名、封地名（可选）
- **谥号字**：1-2个谥号字符
- **爵位**：帝、王、公、侯、伯、子、男之一

#### 示例
- `秦献公` → 前缀:秦, 谥号:献, 爵位:公
- `汉孝文帝` → 前缀:汉, 谥号:孝文, 爵位:帝
- `季桓子` → 前缀:季(姓氏), 谥号:桓, 爵位:子
- `康侯` → 前缀:无, 谥号:康, 爵位:侯

### 4. 前缀验证规则

根据爵位不同，前缀要求不同：

#### 帝、王、公级
- 无前缀限制
- 常见前缀：国名（秦、齐、楚、晋等）

#### 侯、伯、男级
- **重要章节**（本纪、世家、年表）：允许无前缀
  - 如：`康侯`、`庄侯`、`文侯`（出现在年表中）
- **普通章节**：需要有前缀
  - 前缀可以是国名、封地名

#### 子级（大夫）
- 前缀通常是**姓氏**（单字）
  - 常见大夫姓氏：季、孟、叔、仲、高、范、智、赵、韩、魏
  - 如：`季桓子`、`高昭子`、`范献子`
- 完全无前缀的需过滤（除非在重要章节）

### 5. 排除规则

#### 排除思想家/称号
```
文子、列子、庄子、孟子、墨子、荀子、韩非子、商子、申子、尸子、
慎子、田子方、段干子
```
**原因**：这些是学派称号或尊称，非谥号

#### 排除封地侯
```
武安侯、故安侯、清安侯、长平侯、平津侯、安定侯、建成侯、文成侯、
东武侯、令武侯、信武侯、玄武侯
```
**原因**：格式为"封地名+侯"，非谥号

**识别方法**：
- 两字地名 + 侯 → 封地侯（如：武安侯）
- 单字谥号 + 侯 → 可能是谥号（需进一步验证）

### 6. 汉代皇帝特殊规则

汉代皇帝谥号完整形式为"孝X"，简称时省略"孝"字：

| 完整谥号 | 简称 | 说明 |
|---------|------|------|
| 孝文帝 | 文帝 | 同一人 |
| 孝武帝 | 武帝 | 同一人 |
| 孝惠帝 | 惠帝 | 同一人 |
| 孝景帝 | 景帝 | 同一人 |

**处理方式**：
- 索引中分别列出
- 在说明中注明为同一人的不同称呼

### 7. 章节权重分级

不同章节类型对谥号识别有不同的可信度：

#### 高权重章节（42章）
- **本纪**（001-012）：天子、诸侯事迹
- **世家**（031-060）：诸侯国史
- **年表**（013-022）：编年大事记

**特点**：这些章节中的谥号更可靠，允许放宽前缀要求

#### 普通章节
- 列传、书等其他章节
- 谥号识别需要更严格的前缀验证

### 8. 完整识别流程

```
输入：人名实体（如"秦献公"）
  ↓
步骤1：正则匹配 → 提取 [前缀?, 谥号, 爵位]
  ↓
步骤2：验证谥号字 → 是否在谥号字符集中？
  ↓
步骤3：验证爵位 → 是否在七等爵位中？
  ↓
步骤4：检查排除列表 → 是否为思想家/封地侯？
  ↓
步骤5：前缀验证 → 根据爵位和章节类型验证前缀
  ↓
步骤6：章节验证 → 是否出现在高权重章节？
  ↓
输出：谥号识别结果
```

### 9. 数据来源

1. **人名实体索引**: `entity_index.json`
2. **谥法释义**: 《谥法解》等文献
3. **年代数据**: `year_ce_map.json`（可选）
4. **生卒年数据**: `person_lifespans.json`（可选）

### 10. 生成脚本

**脚本位置**: `kg/entities/scripts/build_shihao_index.py`

**主要功能**：
1. 从 entity_index.json 提取所有人名实体
2. 应用谥号识别逻辑筛选谥号人名
3. 按谥号字分组
4. 按爵位等级排序
5. 生成 JSON 数据和 HTML 页面

**输出文件**：
- `kg/entities/data/shihao_index.json` - 结构化数据
- `docs/special/shihao.html` - 可视化页面
