## 根本原因分析

### 1. 规范理解错误

**正确规范**（doc/spec/动词标注规范.md）：
- **动词标注**：必须使用`⟦TYPE动词⟧`格式（U+27E6/U+27E7数学双方括号）
- **实体标注**：使用`〖TYPE内容〗`格式（U+3016/3017白色龟甲括号）
- **动词TYPE符号**：`◈`（军事）、`◉`（刑罚）、`○`（政治）、`◇`（经济）

### 2. 历史遗留问题

项目中存在两种刑法标注格式：

**旧格式**（v2.8及之前）：
```markdown
〖[伐〗、〖[杀〗、〖[诛〗   # 使用实体标注格式
```

**新格式**（v3.0及之后）：
```markdown
⟦◈伐⟧、⟦◉杀⟧、⟦◉诛⟧   # 使用动词标注格式
```

### 3. Agent理解错误

Claude Agent在执行反思任务时：
1. 看到章节中有`⟦◈⟧`/`⟦◉⟧`格式
2. 误认为这是"旧格式"需要"迁移"到`〖[〗`
3. 实际上应该是：保持`⟦◈⟧`/`⟦◉⟧`格式不变，或者将`〖[〗`格式迁移到`⟦◈⟧`/`⟦◉⟧`

## 正确的做法

### 1. 第三轮反思不应该涉及格式迁移

第三轮反思的目标：
- ✅ 检查实体标注遗漏
- ✅ 检查时长消歧
- ✅ 检查人名省称规范
- ✅ 检查原文完整性
- ❌ **不应该**修改动词标注格式

### 2. 如果需要格式迁移

格式迁移应该是单独的任务：
```bash

python kg/entities/scripts/migrate_verb_tags.py --all --backup
```

不应该在反思任务中混入格式迁移工作。
