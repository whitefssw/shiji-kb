#### 数据结构支持

支持两种JSON格式:

**格式1: sections格式** (如 013_三代世表.json):
```json
{
    "sections": [
        {
            "columns": ["列1", "列2", ...],
            "rows": [...]
        }
    ]
}
```

**格式2: rows格式** (如 018_高祖功臣侯者年表.json):
```json
{
    "table_info": {
        "columns": ["列1", "列2", ...]
    },
    "rows": [...]
}
```

#### 问题解决过程

**问题1: JSON链接路径错误**
- 原路径: `../../data/tables/data/*.json`
- 解决: 创建`publish_tables_data.py`复制文件到`docs/special/data/tables/`
- 新路径: `data/tables/*.json`（相对路径）

**问题2: 横向滚动条位置**
- 需求: 横向滚动条固定在表头下方，纵向滚动时始终可见
- 尝试1: `domLayout: 'autoHeight'` + 自定义滚动条（不支持）
- 尝试2: CSS sticky定位ag-Grid滚动条（不生效）
- **最终方案**: 使用`domLayout: 'normal'` + 分页 + CSS sticky定位`.ag-body-horizontal-scroll`

**问题3: 显示行数需求变化**
- 最初: 每页50行
- 调整1: 每页100行
- 调整2: 无分页，全部显示（autoHeight模式）
- **最终**: 恢复分页模式，每页100行（可选），固定高度700px

### 2. 姓氏宗族交互式查询功能开发

**Commit**: [a26af34a](https://github.com/baojie/shiji-kb/commit/a26af34a)

#### 背景

xingshi.html 原本只展示11个先秦大姓的源流信息，缺少具体人物的姓氏数据查询功能。用户需要：
1. 查看《史记》人物的详细姓氏信息（姓、氏、名、字）
2. 支持搜索、筛选、排序功能
3. 可导出数据供研究使用

#### 新增功能

**人物姓氏查询表**（ag-Grid表格组件）:
- **12列详细信息**：
  - 基本信息：人名、姓、氏、名、字、时期
  - 推理信息：姓源、氏类型、氏来源、置信度、推理规则、章节
- **交互功能**：
  - 快速搜索：搜索人名、姓、氏、字等所有字段
  - 列筛选：每列独立筛选器（文本筛选和集合筛选）
  - 排序：单列和多列排序
  - CSV导出：导出当前筛选结果
  - 分页显示：每页100行（可选50/100/200/500）
- **显示优化**：
  - 表头固定在顶部
  - 第一列（人名）固定在左侧
  - 横向滚动条固定在表头下方
  - 表格高度600px
  - 古典配色（#8B4513褐色表头）
- **实时统计**：
  - 总人数、显示人数、先秦人物数

   - 574人更新 + 42人新增 = 2095人

**转换工具开发**:

**主脚本**: `kg/entities/scripts/update_xingshi_from_md.py` (推荐)
- **策略**: MD→JSON 增量更新
- **逻辑**:
  1. 读取现有 JSON（2053人）
  2. 解析 MD 详细表格（616人）
  3. MD 数据覆盖 JSON 对应人物
  4. 保留 JSON 中其他人物数据
  5. 输出合并结果（2095人）
- **优势**: 保留多轮推理结果，不丢失旧数据

**备用脚本**: `kg/entities/scripts/convert_xingshi_md_to_json.py`
- **策略**: MD→JSON 完全转换
- **逻辑**: 仅转换 MD 中的616人详细记录
- **用途**: 当需要全新生成时使用

**发布脚本**: `scripts/publish_xingshi_data.py`
- 自动复制4个文件到 docs 目录
- 统计文件大小和数量
- 简化数据更新流程

#### 建设逻辑文档化

**发现**: person_xingshi.json 的生成采用 **AI Agent 多轮迭代推理** 方法

**核心文档**:
1. **SKILL_07b_姓氏推理.md**: 推理规则和方法论
   - R1: 直接记载（明确的姓氏文本）
   - R2: 邦国推理（根据所属国家推断姓氏）
   - R3: 氏族推理（根据氏族归属）

   - R6-R14: 各种专门规则（章节归属、帝号、谥号等）

2. **prompt_template_姓氏推理.md**: AI Agent 执行的提示词模板
   - 定义每轮迭代的工作流程
   - 输入：已有 JSON + 实体索引 + 章节文本
   - 输出：增量更新的 JSON 条目

3. **执行方式**:
   - 由 AI Agent（如 Claude）按提示词模板执行
   - 每轮迭代新增已知人物，解锁下轮推理
   - 非自动化脚本，需人工/Agent 配合
